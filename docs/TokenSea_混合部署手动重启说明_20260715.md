# TokenSea 混合部署手动重启说明

- 版本：V1.0
- 日期：2026-07-15
- 适用范围：研发环境的“Docker 基础设施 + 本地业务进程”模式。

## 1. 当前研发拓扑

| 运行方式 | 服务 | 端口 |
|---|---|---:|
| Docker | PostgreSQL、Redis、出口代理、Runtime Core | 39213、39214、18080、39218 |
| 本地进程 | Console、Control Plane、Gateway Runtime | 39210、39211、39212 |

本说明不适用于预发布和生产环境；这两个环境应使用完整 Docker Compose。

## 2. 重启前检查

进入项目根目录：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea
```

确认本地配置存在。不要在终端打印 `.env` 内容：

```powershell
Test-Path .\deploy\compose\.env
```

确认 Docker Desktop 已运行：

```powershell
docker info
```

## 3. 日常手动重启（推荐）

### 3.1 停止本地业务进程

在各自运行本地服务的终端按 `Ctrl+C`，依次停止：

```text
Console
Gateway Runtime
Control Plane
```

如果终端已关闭，先确认端口拥有者，再停止确认无误的 TokenSea 进程：

```powershell
Get-NetTCPConnection -LocalPort 39210,39211,39212 -State Listen |
  Select-Object LocalPort, OwningProcess

Get-CimInstance Win32_Process -Filter "ProcessId = <进程ID>" |
  Select-Object ProcessId, Name, CommandLine
```

只有确认命令行为 TokenSea 的 Vite、Control Plane JAR 或 Uvicorn 时，才执行：

```powershell
Stop-Process -Id <进程ID>
```

### 3.2 重启 Docker 基础设施

执行项目脚本。它会移除 Docker 中可能占用 `39210`、`39211`、`39212` 的业务容器，保留数据卷，并启动 PostgreSQL、Redis、出口代理和 Runtime Core：

```powershell
& .\scripts\dev\start-infra.ps1
```

不要执行：

```powershell
docker compose down -v
```

该命令会删除数据库、Redis、Grafana 和 Gateway Outbox 数据卷。

### 3.3 分三个终端启动本地业务进程

终端一：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea
& .\scripts\dev\start-control-plane.ps1
```

Control Plane 使用 JDK 21 打包后运行，启动时会自动执行未应用的 Flyway 迁移。

终端二：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea
& .\scripts\dev\start-gateway.ps1
```

首次运行时，脚本会创建 `services/gateway-runtime/.venv` 并安装 Python 依赖；后续启动不会重复安装。

终端三：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea
& .\scripts\dev\start-console.ps1
```

Console 使用 Vite 热更新；Vue、TypeScript 和 CSS 修改后通常无需重启。

## 4. 验证重启结果

在第四个终端执行：

```powershell
$urls = @(
  'http://localhost:39210',
  'http://localhost:39211/actuator/health',
  'http://localhost:39212/health',
  'http://localhost:39218/health/liveliness'
)
$urls | ForEach-Object {
  $response = Invoke-WebRequest -UseBasicParsing -Uri $_ -TimeoutSec 10
  "$_ => $($response.StatusCode)"
}
```

四个地址均应返回 `200`。

有新的 Flyway 迁移时，查询最近版本：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea\deploy\compose
docker compose -p tokensea --env-file ./.env exec -T tokensea-postgres `
  psql -U tokensea -d tokensea -c `
  "SELECT version, description, success FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"
```

目标迁移必须显示 `success = t`。当前 V14 的预期版本为 `14`。

查看出口代理是否已加载本地 Control Plane 的动态策略：

```powershell
docker compose -p tokensea --env-file ./.env `
  -f docker-compose.yml -f docker-compose.dev.yml `
  logs --tail=50 tokensea-egress-proxy
```

日志出现 `dynamic_policy_refreshed` 即表示刷新成功。

## 5. 特定改动后的额外操作

| 改动类型 | 操作 |
|---|---|
| Vue、TypeScript、CSS | Vite 自动热更新；异常时只重启 Console |
| Java 控制面或 Flyway 迁移 | 重启 Control Plane |
| Gateway Python 代码 | Uvicorn `--reload` 自动重启；异常时重启 Gateway |
| `pom.xml` | 重启 Control Plane，Maven 会解析新增依赖 |
| `requirements.txt` | 删除本地 `.venv` 后重新运行 Gateway 脚本，或在该虚拟环境中安装依赖 |
| 出口代理源码、Dockerfile | 重建出口代理镜像后执行 `start-infra.ps1` |
| Runtime Core 镜像或 Compose 基础设施 | 使用完整 Compose 重建并验证 |

出口代理重建命令：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea\deploy\compose
docker compose -p tokensea --env-file ./.env `
  -f docker-compose.yml -f docker-compose.dev.yml `
  build tokensea-egress-proxy
cd ..\..
& .\scripts\dev\start-infra.ps1
```

## 6. 回退到完整 Docker Compose

先停止三个本地业务进程，然后执行：

```powershell
cd D:\12_其他项目\30_APIGateway\tokensea\deploy\compose
docker compose -p tokensea --env-file ./.env up -d --build
docker compose -p tokensea --env-file ./.env ps
```

完整 Docker 模式下不要叠加 `docker-compose.dev.yml`，否则出口代理会继续访问宿主机本地 Control Plane。

## 7. 常见问题

| 现象 | 处理方式 |
|---|---|
| 端口已被占用 | 查明端口拥有者；只停止确认无误的 TokenSea 进程 |
| Control Plane 启动失败 | 确认 JDK 21 可用；查看终端 Maven/JAR 日志 |
| Gateway 启动失败 | 确认 PostgreSQL、Redis、Runtime Core 健康；检查本地 `.venv` |
| 出口代理反复 `dynamic_policy_refresh_failed` | 确认本地 Control Plane 已在 39211 启动、`TOKENSEA_EGRESS_POLICY_TOKEN` 与 `.env` 一致，并查看代理日志 |
| V14 未出现 | 重启本地 Control Plane，随后查询 `flyway_schema_history` |

自动重启可使用本机技能 `$tokensea-hybrid-restart`；本说明保留人工排障和不使用技能时的可执行步骤。
